### Database Schema (PostgreSQL via Supabase)
**users table:**
- id (UUID, primary key)
- email (unique)
- username
- password_hash (bcryptjs)
- created_at

**trading_profiles table:**
- id (UUID)
- user_id (foreign key to users)
- account_balance
- daily_loss_limit
- daily_loss_used
- strategy_name
- strategy_rules

**chat_messages table:**
- id (UUID)
- user_id (foreign key)
- role ('user' or 'assistant')
- content
- created_at

### Data Isolation ✅
Every user:
- Has their own encrypted trading profile
- Has their own chat history
- Cannot access other users' data
- Authenticated by JWT token on every request

### How It Works Now
1. User signs up → Password hashed, stored in DB
2. Login → JWT token generated, stored in localStorage
3. User chats → Token sent in Authorization header
4. Backend validates token → Loads user's profile from DB
5. Claude evaluates with user's specific rules
6. Chat saved to user's chat history

### Setup Steps for Jackson
1. Set up Supabase (free tier fine): https://supabase.com
2. Copy URL and anon key → add to `.env`
3. Run SQL schema in Supabase dashboard (see AUTH_DATABASE_SETUP.md)
4. `npm install` (install auth dependencies)
5. Create `.env` file with Supabase credentials
6. `npm run dev` (starts backend + frontend)
7. Go to localhost:3000, sign up, start trading!
